# MX Keyboard Switcher — правила для агентов

## Live-тесты X11 и запуск бинаря — ТОЛЬКО на изолированном дисплее

Тесты `crates/mxks-platform/tests/x11_live.rs` и сам демон mxks инжектят
нажатия клавиш через XTEST. На рабочем дисплее (`:10`) эти события попадают в
окно с фокусом — в терминалы пользователя летят фантомные Shift+Tab, Ctrl+C и
буквы, ломая чужие сессии.

Поэтому:

- **Запрещено** запускать `cargo test ... --test x11_live -- --ignored` или
  собранный бинарь mxks с `DISPLAY=:10` (или любым дисплеем реального рабочего
  стола).
- Live-тесты запускать **только** через `scripts/run-x11-live-tests.sh` — он
  поднимает отдельный Xvfb, настраивает раскладки us,ru и выставляет
  `MXKS_TEST_DISPLAY`. Без этой переменной тесты намеренно падают
  (см. `require_isolated_display()` в `tests/x11_live.rs`).
- Для ручного запуска демона в разработке — тоже отдельный Xvfb/Xephyr,
  не рабочий дисплей.
