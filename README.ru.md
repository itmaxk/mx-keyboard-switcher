# MX Keyboard Switcher

Быстрый, лёгкий, кроссплатформенный аналог Punto Switcher на Rust.

Исправляет текст, набранный не в той раскладке: например, вы хотели написать
`привет`, но была включена английская раскладка и получилось `ghbdtn`.
MX Keyboard Switcher распознаёт это, стирает слово, печатает его правильно и
переключает системную раскладку. Работает полностью **офлайн**, потребляет мало
памяти и поставляется одним бинарником на каждую ОС.

> Языки в v1: **русский ↔ английский**. Архитектура основана на таблицах
> раскладок и словарях, поэтому новые пары можно добавить позже.

## Возможности

- **Автоисправление неверной раскладки** (включено по умолчанию):
  `ghbdtn ` → `привет `.
- **Ручная конвертация по горячей клавише**: нажмите **Pause/Break**, чтобы
  преобразовать текущее слово и переключить раскладку. Повторное нажатие (если
  больше ничего не набрано) возвращает слово обратно. Переназначается из трея
  (**Change hotkey → нажмите клавишу**) или в конфиге — сопоставление идёт по
  имени клавиши, поэтому подходит любая клавиша или сочетание.
- **Иконка в трее**: вкл/выкл, переключение автокоррекции, смена горячей клавиши,
  открыть/перезагрузить конфиг, включение автозагрузки.
- **Конфиг TOML** с исключениями по приложениям и словам и своим словарём.
- **Приватность**: никакого доступа к сети; ничто не покидает ваш компьютер.

## Как это работает

Буфер слова хранит **физические клавиши**, а не символы, поэтому конвертация не
зависит от раскладки: одна и та же последовательность клавиш просто отрисовывается
через таблицу другой раскладки. Детекция сочетает поиск по словарю, символьную
биграммную модель языка и эвристику «невозможных сочетаний букв», и сильно
смещена в сторону того, чтобы не трогать текст — ложная замена хуже пропущенной.
Набор регрессионных тестов гарантирует **ноль ложных срабатываний** на тысячах
корректных слов.

## Поддержка платформ

| ОС | Захват | Ввод | Смена раскладки | Примечания |
|----|--------|------|-----------------|------------|
| **Linux (X11)** | XRecord | XTEST | XKB | Полностью работает и протестировано. Wayland — ниже. |
| **Windows** | `WH_KEYBOARD_LL` | `SendInput` | `WM_INPUTLANGCHANGEREQUEST` | Реализовано; проверьте на устройстве. |
| **macOS** | `CGEventTap` | `CGEvent` | TIS | Реализовано; нужен доступ Accessibility. |

### Linux / Wayland

v1 поддерживает **X11** (и X11-приложения под XWayland). Wayland не предоставляет
переносимого API для глобального перехвата/ввода клавиш, поэтому в чистой
Wayland-сессии приложение предупреждает и работает ограниченно. Нативный
Wayland-бэкенд (evdev/uinput + управление раскладкой под каждый композитор)
запланирован.

### macOS

macOS требует разрешение **Accessibility** (Системные настройки →
Конфиденциальность и безопасность → Универсальный доступ), чтобы event tap получал
нажатия. Парольные поля защищены автоматически через Secure Input. На клавиатурах
Mac нет клавиши Pause/Break — задайте другую клавишу `convert_last_word`
(например, `"F13"` или `"Ctrl+Shift+K"`) в конфиге.

### Windows

Низкоуровневый хук клавиатуры может выглядеть как кейлоггер для антивирусов;
помогают подпись бинарника и добавление в исключения. Отсутствие сетевого доступа
— веский аргумент, что программа ничего не передаёт.

## Установка / Сборка

### Быстрая установка (одной командой)

Linux / macOS:

```sh
curl -fsSL https://raw.githubusercontent.com/itmaxk/mx-keyboard-switcher/main/scripts/install.sh | sh
```

Windows (PowerShell):

```powershell
powershell -ExecutionPolicy Bypass -c "irm https://raw.githubusercontent.com/itmaxk/mx-keyboard-switcher/main/scripts/install.ps1 | iex"
```

Флаги:

- Linux/macOS: `sh -s -- --no-autostart` · `--from-source` · `--version vX.Y.Z`
- Windows (с флагами через пайп): `& ([scriptblock]::Create((irm https://raw.githubusercontent.com/itmaxk/mx-keyboard-switcher/main/scripts/install.ps1))) -NoAutostart`

Если для вашей платформы ещё нет релиза, скрипт соберёт бинарь из исходников (нужен `cargo`).


Нужен свежий тулчейн Rust (`rustup`, stable).

```sh
# Зависимость сборки трея на Linux (StatusNotifierItem через DBus):
sudo apt-get install -y libdbus-1-dev pkg-config    # Debian/Ubuntu

# Сборка релизного бинарника:
cargo build --release
# Бинарник: target/release/mx-keyboard-switcher

# Сборка без трея (libdbus не нужен):
cargo build --release --no-default-features
```

Запуск:

```sh
./target/release/mx-keyboard-switcher
# Подробные логи:
MXKS_LOG=debug ./target/release/mx-keyboard-switcher
```

## Настройка

При первом запуске создаётся конфиг с комментариями:

- Linux: `~/.config/mx-keyboard-switcher/config.toml`
- macOS: `~/Library/Application Support/mx-keyboard-switcher/config.toml`
- Windows: `%APPDATA%\mx-keyboard-switcher\config.toml`

```toml
[general]
autocorrect = true        # автоисправление неверной раскладки
min_word_len = 3          # игнорировать слова короче этого

[hotkeys]
convert_last_word = "Pause"   # или "Ctrl+Shift+K", "F13", ...

[detection]
threshold = 3.0           # больше = консервативнее (меньше замен)

[exclusions]
apps = ["keepassxc", "1password"]   # подстроки имён приложений для пропуска
words = []                          # набранные формы, которые не исправлять

[dictionary]
extra_en = []             # дополнительные корректные слова (не исправляются)
extra_ru = []
```

Отредактируйте файл и выберите **Reload config** в трее (смена горячей клавиши
требует перезапуска).

## Ограничения (v1)

- Автокоррекция срабатывает по разделителю **пробел**; Enter/Tab/пунктуация
  завершают слово, но не запускают автозамену (ручная клавиша работает).
- Исключения по приложениям и детекция парольных полей — по возможности и пока
  в первую очередь на Linux.
- Wayland — только X11/XWayland (см. выше).
